Runtime stack model:

Every function call creates a stack frame which consists of local variable data and the return address. The program counter is changed to the function code during the function call and to the return address in the stack when the return command is executed in the function code. During the execution of the function, only the local variables in the stack frame can be accessed.

When the return instruction is executed, the stack frame is destroyed.

The Stack Pointer is a register that points to the top (bottom actually) of the stack, i.e. to the lowest address in the stack. The stack grows downwards, adding new stack frames to lower addresses in the stack.

DON’T call a function before you declare it. Because the compilation happens line by line. If the compiler doesn’t know where the function code is to be located in the executable, it can’t make the execution jump to that address during the function call. A way to solve this is to declare the function name before the call, but defining it later. Example:

int func(int num);
int main(){
	int x = func(2);
	//stuff...
	return 0;
}
int func(int num){
	//stuff..
	return //stuff
}

Pass by value is not helpful here:

#include<iostream>
void swap(int x, int y){
	int z = y;
	y = x
	x = z
	// we expect this function to swap the values of x and y
}
int main(){
	int x = 3;
	int y = 4;
	swap(x,y);
	std::cout << x <<< " " << y << endl; //this should print 4 3 right?
	// Well.. since the x and y in the function body refer to the local variables stored in the stack frame,
	// the global x and y were unaffected.
	return 0;
}

Pass by reference:

#include<iostream>
void swap(int& x, int& y){
	int z = y;
	y = x
	x = z
	// we expect this function to swap the values of x and y
}
int main(){
	int a = 3;
	int b = 4;
	swap(a,b);
	std::cout << a <<< " " << b << endl; //this should print 4 3 right?
	// Yes, this actually does the thing.
	return 0;
}

Basic Quadratic Solver function using pass by reference:

void QS(int a, int b, int c, double& x1, double& x2){
	int D = b*b - 4*a*c; 
	if(D < 0){
		cout<<"No solution"<<endl;
	} else if(D==0){
		x1 = - (double)b / (2 * (double)a);
		cout<<"One solution ; stored in x1"<<endl;
	} else {
		double B = - (double) b;
		double A = (double) (2*a);
		double K = pow(((double) D),0.5);
		x1 = (B+K)/A;
		x2 = (B-K)/A;
		cout<<"Two Solutions"<<endl;
	}
}

Function Overloading:

You can have multiple functions with the same name, as long as their return type and arguments aren’t the same.

#include<iostream>
using namespace std;
int Pluss(int x,int y){
	return x + y;
}
double Pluss(double x,double y){
	return x + y;
}
int main(){
	cout << Pluss(2,3) << endl;
	cout << Pluss(2.0,3.0) << endl;
	return 0;
}
